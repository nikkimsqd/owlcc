Call Center Flow


Specifications

  1. A Twilio phone number is configured with an IVR.
  2. The IVR receives incoming callers, says a message, and adds the caller into a TaskRouter managed TaskQueue.
    While in the queue, callers listen to music.
  3. There is a single TaskRouter Workspace:
    1. One TaskQueue
    2. One Workflow
    3. Worker agents
    4. Agent Client call Activities, for example: offline, idle (available), and busy.
  4. Agents connect to the Agent Application with their web browser and set their status activity to idle, which means they are available to receive calls.
    About the application:
    1. It can either be deployed to Heroku, run on a localhost computer, or any other computer which is PHP enabled.
    2. The application is a simplified version of the Twilio Call Center training course application.
